# /quick-fix - バグ修正ワークフロー

バグを素早く修正するためのガイド付きワークフローです。

## 問題の説明

$ARGUMENTS

## 修正ワークフロー

### Step 1: 問題の特定

1. エラーメッセージやログを確認
2. 再現手順を特定
3. 関連するコードを探す

```bash
# エラーメッセージでコード検索
grep -r "エラーメッセージ" src/
```

### Step 2: 原因の調査

1. 関連ファイルを読む
2. データフローを追跡
3. エッジケースを確認

**レイヤー構造を意識:**
- `handlers/` → 入力バリデーション
- `services/` → ビジネスロジック
- `utils/` → 外部API、ユーティリティ

### Step 3: 修正の実装

1. 最小限の変更で修正
2. 型安全性を維持（Zodバリデーション使用）
3. エラーハンドリングを適切に

### Step 4: テスト

```bash
# 関連テストを実行
npm test

# 型チェック
npm run typecheck

# lint チェック
npm run lint
```

### Step 5: 動作確認

```bash
# ビルドして動作確認
npm run dev
```

### Step 6: コミット

修正が完了したら、適切なコミットメッセージでコミット：

```
fix: 問題の簡潔な説明

- 修正内容1
- 修正内容2
```

## 注意点

- 修正は最小限に（他の部分を触らない）
- テストを追加/更新する
- 既存の動作を壊さないように注意
